## Step 4 â€” wire GHCR for prod

The `release.yml` workflows publish on `push: main` and on tag pushes (`v*.*.*`). They use `${{ github.event.repository.name }}`, so they automatically publish under your fork's path: `ghcr.io/<your-owner>/<your-renamed-repo>:latest`. No edit needed.

The default `GITHUB_TOKEN` has `packages: write` permission via the workflow-level grant, so the first push from a freshly-forked repo works without any secret setup.

**Make the GHCR package public** the first time the workflow publishes:

1. Go to `https://github.com/<your-owner>/<your-renamed-repo>/pkgs/container/<your-renamed-repo>`
2. Package settings â†’ "Change visibility" â†’ Public.

Otherwise downstream consumers (your Hetzner VPS) need a pull credential.

## Step 5 â€” wire prod compose to your images

In `acme-infra/compose/.env`:

```bash
# If you renamed your repos:
IMAGE_OWNER=acme
API_IMAGE_NAME=acme-api
UI_IMAGE_NAME=acme-ui
```

If you kept the canonical names, only `IMAGE_OWNER` needs to change.

## Step 6 â€” wire OpenTofu bootstrap

In `acme-bootstrap/terraform.tfvars`:

```hcl
api_repo   = "https://github.com/acme/acme-api"
ui_repo    = "https://github.com/acme/acme-ui"
infra_repo = "https://github.com/acme/acme-infra"
domain     = "acme.com"
# ... other vars per terraform.tfvars.example
```

The bootstrap module derives `IMAGE_OWNER`, `API_IMAGE_NAME`, and `UI_IMAGE_NAME` from those URLs and renders them into `compose/.env` on the VPS automatically. You never set them twice.

## Checklist

- [ ] All four template repos forked under your org/user
- [ ] First image successfully published to GHCR (push to `main` and watch the Actions tab)
- [ ] GHCR package visibility set to Public
- [ ] `acme-infra/compose/.env` set with your `IMAGE_OWNER` (+ `*_IMAGE_NAME` if renamed)
- [ ] `acme-bootstrap/terraform.tfvars` set with your fork URLs and a real `domain`
- [ ] `demo@example.com` / `password123` deleted or rotated before any production traffic
- [ ] `JWT_SECRET` regenerated for production
- [ ] `ACME_EMAIL` set to a real address (not `example.com`)
