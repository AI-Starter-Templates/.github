The api-template ships [Cloudflare Email Service](https://developers.cloudflare.com/email-service/) as the default outbound mail provider. Combined with Workers Paid ($5/mo, no documented per-message charge), it's the cheapest transactional email path that exists at non-trivial volume.

## Why

|                  | Cloudflare Email Service                              | Resend                 | SendGrid           |
| ---------------- | ----------------------------------------------------- | ---------------------- | ------------------ |
| Free tier        | Bundled into Workers Paid ($5/mo)                     | 3,000/mo, 100/day      | 100/day forever    |
| Per-message cost | Effectively zero (beta)                               | $20 / 50k emails (Pro) | $20 / 50k emails   |
| Setup            | Cloudflare-managed DNS auto-provisions SPF/DKIM/DMARC | Manual DNS records     | Manual DNS records |
| Vendor lock-in   | Cloudflare account required                           | None                   | None               |

If you're already paying for Workers (or willing to), Cloudflare Email is **free transactional sending** at the margin. That maps directly to BoringStack's cost-of-goods bias.

## How the api-template uses it

The api-template has a pluggable email provider abstraction (`src/lib/email/`). Each provider implements the same `IEmailService` interface:

```ts
export interface IEmailService {
  send: (message: IEmailMessage) => Promise<IEmailResult>;
  readonly providerName: EmailProviderName;
}
```

`buildEmailService()` reads `EMAIL_PROVIDER` from env and constructs the right concrete provider. When it's `cloudflare`, the implementation POSTs to:

```
POST https://api.cloudflare.com/client/v4/accounts/{CLOUDFLARE_ACCOUNT_ID}/email/sending/send
Authorization: Bearer {CLOUDFLARE_EMAIL_API_TOKEN}
Content-Type: application/json
```

with `{from, to, subject, html, text?}`. Retries + structured logging + the shared `retryWithBackoff` helper are reused — Cloudflare gets the same robustness wrap as Resend and SendGrid.

## Quickstart (dev)

In dev with both Cloudflare env vars empty, a noop provider logs the rendered email payload to stdout instead of sending. No friction.

To actually send in dev (rare), set:

```bash
EMAIL_PROVIDER=cloudflare        # already the default
EMAIL_FROM=noreply@<your-cf-domain>
CLOUDFLARE_ACCOUNT_ID=<from CF dashboard right sidebar>
CLOUDFLARE_EMAIL_API_TOKEN=<API token with "Email Sending: Edit">
```

## Switching providers

Same env switch, different keys:

```bash
EMAIL_PROVIDER=resend
RESEND_API_KEY=re_xxx
```

```bash
EMAIL_PROVIDER=sendgrid
SENDGRID_API_KEY=SG.xxx
```

In production the env validator demands the matching key for the selected provider; the boot fails with a readable error if it's missing.

## Caveats

- **Cloudflare Email Service is in beta** (as of writing). Pricing may evolve; daily limits are variable + account-scoped.
- **DNS must be on Cloudflare.** The auto-provisioned SPF/DKIM/DMARC records are how deliverability works; you can't skip that step.
- **New accounts** can only send to verified addresses until upgraded. The runbook covers the unlock.
