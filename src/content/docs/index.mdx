## What we optimize for

<CardGrid>
  <Card title="Cost to run (COGS)" icon="approve-check">
    Biased toward open-source, self-hosted spine: Postgres, Redis,
    Traefik. Cloudflare Email Service as the default mail provider for
    near-zero per-message cost. Your data plane runs on primitives you
    control.
  </Card>
  <Card title="Lint that backs up the folders" icon="puzzle">
    Fifteen custom ESLint plugins codify the architecture so "where does
    this go?" is a mechanical question. Agents skip `AGENTS.md`; lint
    still enforces the rules.
    <br /><a href="/architecture/lint-as-contract/">See the inventory →</a>
  </Card>
  <Card title="Boots on your machine" icon="bars">
    `./dev.sh up -d` brings up the whole stack: Postgres, Redis, Traefik,
    migrations, demo user seeded. Sign in at `http://localhost:3001` with
    `demo@example.com` / `password123` and you're in.
  </Card>
  <Card title="Typed at every layer" icon="setting">
    OpenAPI from the API, codegen on the UI side, env validators on both
    sides, strict TS everywhere. Drift between server and client is a
    compile error.
  </Card>
</CardGrid>
