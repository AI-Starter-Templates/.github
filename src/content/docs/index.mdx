    <div class="bs-section-heading">
      <p class="bs-eyebrow">The composed-stack bet</p>
      <h2 id="bs-compose-title">Not a starter kit. A product spine.</h2>
      <p>Most templates give you a UI and a login form. BoringStack gives you the seams that usually appear after launch: account boundaries, session rotation, idempotent jobs, webhooks, audit trails, deploy scripts, backups, and local services that match production closely enough to matter.</p>
    </div>

    <div class="bs-story-list">
      <a href="/api/overview/">
        <span>API</span>
        <strong>Auth, billing, queues, email, and audit are already wired.</strong>
        <small>Bun, Elysia, Drizzle, Postgres, Valkey, BullMQ, Stripe hooks, OAuth, and transactional email form the backend skeleton of a real SaaS.</small>
      </a>
      <a href="/ui/overview/">
        <span>UI</span>
        <strong>The React app is typed against the API, not hand-waved.</strong>
        <small>Vite, TanStack Query, shadcn/ui, Playwright, i18n, and a generated OpenAPI client keep server and client changes moving together.</small>
      </a>
      <a href="/infra/overview/">
        <span>Infra</span>
        <strong>Local Compose, production Compose, optional VPS bootstrap.</strong>
        <small>Postgres, Valkey, Traefik, observability overlays, Mailpit, GlitchTip, WUD, backups, and OpenTofu are arranged as one deployable shape.</small>
      </a>
    </div>
  </section>
