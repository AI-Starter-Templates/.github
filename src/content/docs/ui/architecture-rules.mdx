The ui-template enforces its architecture through ESLint, not vibes. The UI-specific plugin is [`eslint-plugin-react-component-architecture`](https://github.com/agjs/eslint-plugin-react-component-architecture) (15 rules: component anatomy, hook placement, `className` discipline, file-naming, prop ordering). It composes with the five shared plugins (resource-architecture, module-boundaries, structured-logging, env-access, test-conventions) that both repos use.

The full inventory across api-template and ui-template (including why machine-checked rules beat prose docs for agent-driven work) lives at [Lint as the contract](/architecture/lint-as-contract/).
