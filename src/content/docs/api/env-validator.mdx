title: Env validator

import { Aside } from "@astrojs/starlight/components";

The api-template validates every env var at boot via a TypeBox schema. Cross-field invariants (e.g. "if `EMAIL_PROVIDER=resend` then `RESEND_API_KEY` is required") are encoded as refinements. The process refuses to start if anything is missing or malformed. There are no silent fallbacks.
