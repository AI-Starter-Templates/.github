Local stack up in about five minutes. Postgres, Valkey, API, UI, all in Docker, demo user pre-seeded. The dev stack uses Vite's dev-server to proxy `/api/*` to api-dev, so the browser sees one origin and there are no CORS preflights.

## Prerequisites

- Docker + Docker Compose v2 (`docker compose version` reports `v2.x` or newer).
- About 4 GB of free RAM.
- No local Node or Bun needed; everything runs in containers.

## Clone four siblings

The four templates live as siblings under one parent. Click "Use this template" on each GitHub repo (see the [after-fork checklist](/topics/use-this-template/)), or clone the upstream:

```bash
mkdir boringstack && cd boringstack

git clone https://github.com/AI-Starter-Templates/api-template
git clone https://github.com/AI-Starter-Templates/ui-template
git clone https://github.com/AI-Starter-Templates/infra-docker-compose-template
```

Compose build contexts use `../../api-template` and `../../ui-template`, so the directory names matter.

<Aside type="tip" title="Optional: setup.sh">
  The workspace root ships a `setup.sh` that bootstraps `compose/.env`,
  generates a `GLITCHTIP_SECRET_KEY`, and (with `--up`) boots the stack.
  Skip if you prefer doing the steps by hand.
</Aside>

## Boot

<Steps>

1. Seed the env:

   ```bash
   cd infra-docker-compose-template/compose
   cp .env.example .env
   ```

2. Make scripts executable:

   ```bash
   chmod +x dev.sh ../scripts/*.sh
   ```

3. Run:

   ```bash
   ./dev.sh up -d --build
   ```

   First boot pulls base images, builds the api/ui dev images, runs migrations, seeds the demo user. About 3 minutes on a fast laptop.

</Steps>

## What's running

| Service | Host port | What it does |
| --- | --- | --- |
| `postgres` | 5432 | App database, schemas `auth`, `billing`, `audit`, `app` |
| `valkey` | 6379 | Cache + BullMQ queues |
| `api-migrate` | (one-shot) | `db:push` + seed demo user, then exits |
| `api-dev` | 3000 | Bun + Elysia API, hot reload via bind mount |
| `ui-dev` | 3001 | Vite dev server, hot reload, proxies `/api/*` to `api-dev` |

Traefik runs only in the prod profile. In prod it terminates TLS and path-routes `/api/*` + `/health` to the api container, everything else to ui, on one domain.

## Stop the stack

| Task | Command | Data preserved? |
| --- | --- | --- |
| Stop, keep data | `./scripts/compose-down.sh` | Yes |
| Stop and wipe data | `./scripts/compose-down-clean.sh` | No (prompts; `CONFIRM=yes` to skip) |
