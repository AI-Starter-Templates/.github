## TL;DR

1. Click **"Use this template"** on the API, UI, and infra repos under your org.
2. Clone your forks as siblings under one parent directory.
3. `cd <infra-fork> && cp compose/.env.example compose/.env && ./scripts/compose-up.sh`
4. Open `http://localhost:3001` and sign up.

You now have Postgres, Valkey, the API, and the UI running locally with hot reload. The optional OpenTofu repo is only needed when you want repeatable first-time VPS provisioning.

## Prerequisites

- Docker + Docker Compose v2 (`docker compose version` reports `v2.x` or newer).
- About 4 GB of free RAM.
- No local Node or Bun needed; everything runs in containers.

## Step 1. Use the templates

BoringStack's local development path needs three GitHub template repos. On each repo, click **"Use this template"**, then **"Create a new repository"**:

- [`api-template`](https://github.com/AI-Starter-Templates/api-template), e.g. `acme-api`
- [`ui-template`](https://github.com/AI-Starter-Templates/ui-template), e.g. `acme-ui`
- [`infra-docker-compose-template`](https://github.com/AI-Starter-Templates/infra-docker-compose-template), e.g. `acme-infra`

Fork [`infra-bootstrap-tofu-template`](https://github.com/AI-Starter-Templates/infra-bootstrap-tofu-template) later if you want `tofu apply` to provision your first VPS.

Keep the canonical names if you want. Nothing breaks either way.

<DocCallout variant="note" title="Skip the docs repo" eyebrow="Template boundary">
The docs site source (`.github` repo) is **not** a template. It's BoringStack-specific content. Skip it.
</DocCallout>

## Step 2. Clone your forks as siblings

The compose build contexts use relative paths (`../../api-template`, `../../ui-template`). Whatever you named your forks, lay them out as siblings under one parent:

```bash
mkdir acme && cd acme

git clone https://github.com/<you>/acme-api
git clone https://github.com/<you>/acme-ui
git clone https://github.com/<you>/acme-infra
# Optional, for later VPS provisioning:
git clone https://github.com/<you>/acme-bootstrap
```

If you renamed `api-template` or `ui-template`, either match the cloned directory name to what `compose/docker-compose.yml` expects, or edit the `build: context:` lines once.

<DocCallout variant="tip" title="Optional: setup.sh" eyebrow="Fast path">
The workspace root ships a `setup.sh` that bootstraps `compose/.env`, generates a `GLITCHTIP_SECRET_KEY`, and (with `--up`) boots the stack. Skip if you prefer doing the steps by hand.
</DocCallout>

## Step 3. Boot the local stack

<Steps>

1. Seed the env:

   ```bash
   cd acme-infra
   cp compose/.env.example compose/.env
   ```

2. Make scripts executable:

   ```bash
   chmod +x compose/dev.sh scripts/*.sh
   ```

3. Run:

   ```bash
   ./scripts/compose-up.sh up -d --build
   ```

   First boot pulls base images, builds the api/ui dev images, runs migrations. If you set `SUPERUSER_EMAIL` + `SUPERUSER_PASSWORD` in `compose/.env`, an admin user is also created. About 3 minutes on a fast laptop.

</Steps>

## Step 4. Sign in

Open **http://localhost:3001**.

If you set `SUPERUSER_EMAIL` + `SUPERUSER_PASSWORD` in `compose/.env` before booting, sign in with those credentials. Otherwise hit "Sign up" on the form and register a new account; that user is the first user in the system. You land on the dashboard, which is intentionally minimal. Start building from here.

## What's running

<FaqGroup>
  <FaqItem title="postgres (port 5432)" open>
    App database; schemas `auth`, `billing`, `audit`, `app`, `notifications`.
  </FaqItem>
  <FaqItem title="valkey (port 6379)">
    Cache and BullMQ queues.
  </FaqItem>
  <FaqItem title="api-migrate (one-shot)">
    `db:push` and optional superuser seed, then exits.
  </FaqItem>
  <FaqItem title="api-dev (port 3000)">
    Bun + Elysia API with hot reload via bind mount.
  </FaqItem>
  <FaqItem title="ui-dev (port 3001)">
    Vite dev server; proxies `/api/*` to `api-dev`.
  </FaqItem>
</FaqGroup>

Traefik runs only in the prod profile. In prod it terminates TLS and path-routes `/api/*` + `/health` to the api container, everything else to ui, on one domain.

<FaqGroup>
  <FaqItem title="Stop, keep data" open>

  </FaqItem>
  <FaqItem title="Stop and wipe data">
