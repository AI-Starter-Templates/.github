The infra-docker-compose-template composes the sibling repos into one running stack. Single-host VPS focus (Hetzner / OVH / DO / your laptop); a Kubernetes-flavored variant is planned as a separate repo.

## Service inventory

The base stack (always on) plus per-profile services:

<FaqGroup>
  <FaqItem title="postgres (always)" open>App DB.</FaqItem>
  <FaqItem title="valkey (always)">Cache + queues (Redis-protocol; Valkey fork).</FaqItem>
  <FaqItem title="traefik (prod)">Reverse proxy + ACME; path-routes one apex domain.</FaqItem>
  <FaqItem title="api-migrate (one-shot)">Schema push + optional superuser seed.</FaqItem>
  <FaqItem title="api-dev (dev)">Bun + Elysia API, bind-mounted source.</FaqItem>
  <FaqItem title="ui-dev (dev)">Vite dev server proxying to api-dev.</FaqItem>
  <FaqItem title="api (prod)">API image from GHCR.</FaqItem>
  <FaqItem title="ui (prod)">UI image from GHCR (nginx static).</FaqItem>
</FaqGroup>

## Profile model

```bash
./dev.sh up -d                                 # baseline dev (no Traefik; Vite proxies /api/*)
STACK=prod ./dev.sh up -d                      # production: GHCR images, Traefik, HTTPS via ACME
WITH_OBSERVABILITY=1 ./dev.sh up -d            # + Prometheus/Grafana/Loki/Promtail/exporters
WITH_GLITCHTIP=1 ./dev.sh up -d                # + self-hosted error tracking
WITH_BULLMQ=1 ./dev.sh up -d                   # + queue dashboard at bullmq.localhost
WITH_WUD=1 ./dev.sh up -d                      # + image-update detection (Discord webhook)
WITH_MAILPIT=1 ./dev.sh up -d                  # + local SMTP catcher at :8025 (dev only)
```

Flags compose: `WITH_OBSERVABILITY=1 WITH_GLITCHTIP=1 ./dev.sh up -d` brings up both overlays alongside the base. See [Profiles & overlays](/infra/profiles-and-overlays/).

## Networks

Two bridge networks. Services are intentionally split:

<FaqGroup>
  <FaqItem title="backend network" open>
    postgres, valkey, api[-dev], api-migrate, exporters, glitchtip, mailpit
  </FaqItem>
  <FaqItem title="frontend network">
    traefik (prod only), ui[-dev], glitchtip-web
  </FaqItem>
</FaqGroup>

In prod, Traefik joins both networks so it can route HTTPS from `frontend` to services on `backend`. The data plane (Postgres, Valkey) is never exposed to `frontend`.

## Resource budgets

Every service has `deploy.resources.limits` and `reservations` driven by env vars with sane defaults sized for a 4-vCPU / 8 GB host. Adjust in `compose/.env`; see [Resource limits](/infra/resource-limits/) for sizing guidance.

## What's not in this template

- Kubernetes manifests. Planned as a separate repo. This template is Compose-first; mixing Compose and cluster YAML in one repo is exactly the kind of confusion BoringStack avoids.
- Application code. The api-template and ui-template own that.
- Cloud-provider provisioning. A separate [OpenTofu template](/topics/provisioning-with-tofu/) handles that.
