## Networks

Two bridge networks. Services are intentionally split:

<SignalGrid
  columns={2}
  items={[
    {
      label: "backend",
      title: "Data plane network",
      body: "postgres, valkey, api[-dev], api-migrate, exporters, glitchtip, and mailpit.",
    },
    {
      label: "frontend",
      title: "Ingress network",
      body: "traefik in prod, ui[-dev], and glitchtip-web. Traefik joins both when it needs to route.",
    },
  ]}
/>

In prod, Traefik joins both networks so it can route HTTPS from `frontend` to services on `backend`. The data plane (Postgres, Valkey) is never exposed to `frontend`.

## Resource budgets

Every service has `deploy.resources.limits` and `reservations` driven by env vars with sane defaults sized for a 4-vCPU / 8 GB host. Adjust in `compose/.env`; see [Resource limits](/infra/resource-limits/) for sizing guidance.

## What's not in this template

- Kubernetes manifests. Planned as a separate repo. This template is Compose-first; mixing Compose and cluster YAML in one repo is exactly the kind of confusion BoringStack avoids.
- Application code. The api-template and ui-template own that.
- Cloud-provider provisioning. A separate [OpenTofu template](/topics/provisioning-with-tofu/) handles that.
