## Infra (docker-compose stack)

| Task | Command | Data preserved? |
|---|---|---|
| Start dev stack | `./scripts/compose-up.sh` | n/a |
| Stop, keep data | `./scripts/compose-down.sh` | ✅ |
| Stop and wipe data | `./scripts/compose-down-clean.sh` | ❌ (prompts; `CONFIRM=yes` to skip) |
| Start prod | `STACK=prod ./scripts/compose-up.sh` | n/a |
| With observability | `WITH_OBSERVABILITY=1 ./scripts/compose-up.sh` | n/a |
| With error tracking | `WITH_GLITCHTIP=1 ./scripts/compose-up.sh` | n/a |
| With queue dashboard | `WITH_BULLMQ=1 ./scripts/compose-up.sh` | n/a |
| With image-update bot | `WITH_WUD=1 ./scripts/compose-up.sh` | n/a |
| Service status | `docker compose -f compose/docker-compose.yml -f compose/docker-compose.development-labels.yml --profile dev ps` | n/a |
| Live logs | `docker compose ... logs -f api-dev` | n/a |
| Live resource use | `docker stats` | n/a |
| Backup db (offsite) | `BACKUP_DRY_RUN=1 ./scripts/backup-wrapper.example.sh` then drop the flag | ✅ |

Flags compose: `WITH_OBSERVABILITY=1 WITH_GLITCHTIP=1 ./scripts/compose-up.sh`.

## Inside the dev containers (rare)

```bash
# Shell into api-dev
docker compose exec api-dev sh
# Tail just one service's logs
docker compose logs -f --tail=100 api-dev
# Connect to Postgres
docker compose exec postgres psql -U app -d app
# Connect to Redis
docker compose exec redis redis-cli
```
