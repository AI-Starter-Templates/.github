Each repo has its own `.env.example` that is the **source of truth** for vars it consumes. This page is an index; it tells you which repo owns a given var, why it exists, and which `.env.example` to read for the full reference (defaults, comments, prod-only flags).

## Which file to edit

<FaqGroup>
  <FaqItem title="api-template: .env in api-template/" open>
    Consumed by the Bun process directly when running outside Docker.
  </FaqItem>
  <FaqItem title="ui-template: .env.local in ui-template/">
    Vite at build and dev time (`VITE_*` only).
  </FaqItem>
  <FaqItem title="infra (dev): compose/.env in infra-docker-compose-template/">
    Compose: injected into the api and ui dev containers.
  </FaqItem>
  <FaqItem title="infra (prod): compose/.env + compose/api.prod.env">
    Compose: prod profile uses a separate api env file.
  </FaqItem>
</FaqGroup>

For Docker Compose-driven setups (the default), **`compose/.env` is the single file you edit.** It overrides defaults baked into `docker-compose.yml` and gets injected into the right containers via `env_file:` references.

## Categories

### Process + observability

<FaqGroup>
  <FaqItem title="NODE_ENV" open>
    **Repo:** api. **Required:** yes. Values `development`, `production`, or `test`; gates prod-only invariants.
  </FaqItem>
  <FaqItem title="PORT">
    **Repo:** api. **Required:** no. Default 3000.
  </FaqItem>
  <FaqItem title="APP_NAME">
    **Repo:** api. **Required:** no (default `API Template`). Public-facing identity used in email From-names, OAuth consent metadata, and the OpenAPI document title.
  </FaqItem>
  <FaqItem title="LOG_LEVEL">
    **Repo:** api. **Required:** no. One of `debug`, `info`, `warn`, `error`.
  </FaqItem>
  <FaqItem title="SENTRY_DSN">
    **Repo:** api. **Required:** no. Empty means the SDK is a no-op.
  </FaqItem>
  <FaqItem title="SENTRY_TRACES_SAMPLE_RATE">
    **Repo:** api. **Required:** no. Range 0 to 1, default 0.1.
  </FaqItem>
  <FaqItem title="VITE_SENTRY_DSN">
    **Repo:** ui. **Required:** no. Same wire protocol as Sentry or GlitchTip.
  </FaqItem>
</FaqGroup>

### Database + cache

<FaqGroup>
  <FaqItem title="DATABASE_URL" open>
    **Repo:** api. **Required:** yes. Postgres connection string.
  </FaqItem>
  <FaqItem title="DATABASE_SSL_REJECT_UNAUTHORIZED">
    **Repo:** api. **Required:** no. Default `true`. Keep it true in production so Postgres TLS is verified instead of merely encrypted.
  </FaqItem>
  <FaqItem title="DATABASE_SSL_CA">
    **Repo:** api. **Required:** no. Optional PEM CA bundle for production Postgres providers that use a private CA.
  </FaqItem>
  <FaqItem title="VALKEY_HOST / VALKEY_PORT">
    **Repo:** api. **Required:** yes when queues or cache use Valkey. Defaults `localhost` and 6379.
  </FaqItem>
  <FaqItem title="VALKEY_PASSWORD">
    **Repo:** api. **Required:** in production when queues, Valkey cache, notification SSE, or OAuth is on.
  </FaqItem>
  <FaqItem title="VALKEY_DB">
    **Repo:** api. **Required:** no. Default 0.
  </FaqItem>
  <FaqItem title="CACHE_ENABLED / CACHE_PROVIDER">
    **Repo:** api. **Required:** no. `false` uses noop; providers include `memory` and `valkey`.
  </FaqItem>
  <FaqItem title="QUEUES_ENABLED">
    **Repo:** api. **Required:** no. When `false`, dispatch falls back to inline execution.
  </FaqItem>
  <FaqItem title="NOTIFICATIONS_SSE_ENABLED">
    **Repo:** api. **Required:** no. Enables the live notification stream. `true` requires Valkey because API instances coordinate through pub/sub.
  </FaqItem>
</FaqGroup>

### Auth + session

<FaqGroup>
  <FaqItem title="JWT_SECRET" open>
    **Repo:** api. **Required:** yes (32+ chars). Signs the 15-minute access cookie and hashes refresh tokens. Rotation forces every user to sign in again.
  </FaqItem>
  <FaqItem title="FRONTEND_URL">
    **Repo:** api. **Required:** yes. App origin used for OAuth success redirects and Stripe return-url allowlisting.
  </FaqItem>
  <FaqItem title="PUBLIC_API_URL">
    **Repo:** api. **Required:** yes. Public URL used to build OAuth callback URLs. Same-origin default is the frontend origin; cross-origin setups use the API origin.
  </FaqItem>
  <FaqItem title="ALLOWED_ORIGINS">
    **Repo:** api. **Required:** no (default empty). CSV. Empty means same-origin deployment (default); CORS is not mounted. When set, every origin must be HTTPS with no wildcards.
  </FaqItem>
  <FaqItem title="SUPERUSER_EMAIL / SUPERUSER_PASSWORD">
    **Repo:** api. **Required:** no (default empty). Optional first-boot superuser. Both set means `db:seed` creates an `admin` user on first run. Empty means no-op; register via the UI instead.
  </FaqItem>
  <FaqItem title="GOOGLE_OAUTH_CLIENT_ID / GOOGLE_OAUTH_CLIENT_SECRET">
    **Repo:** api. **Required:** when Google OAuth is enabled.
  </FaqItem>
  <FaqItem title="GITHUB_OAUTH_CLIENT_ID / GITHUB_OAUTH_CLIENT_SECRET">
    **Repo:** api. **Required:** when GitHub OAuth is enabled.
  </FaqItem>
  <FaqItem title="LINKEDIN_OAUTH_CLIENT_ID / LINKEDIN_OAUTH_CLIENT_SECRET">
    **Repo:** api. **Required:** when LinkedIn OAuth is enabled.
  </FaqItem>
  <FaqItem title="RATE_LIMIT_MAX / RATE_LIMIT_WINDOW_MS">
    **Repo:** api. **Required:** no (defaults 100 / 60000ms). Per-IP global rate limit; defence-in-depth alongside Traefik's edge limit.
  </FaqItem>
  <FaqItem title="AUTH_RATE_LIMIT_MAX / AUTH_RATE_LIMIT_WINDOW_MS">
    **Repo:** api. **Required:** no (defaults 10 / 60000ms). Stricter per-IP limit applied to every `/api/v1/auth/*` route. Tightens the global default for the credential-stuffing surface.
  </FaqItem>
</FaqGroup>

### Email

<FaqGroup>
  <FaqItem title="EMAIL_PROVIDER" open>

  </FaqItem>
  <FaqItem title="EMAIL_FROM">
    **Repo:** api. **Required:** always. Sender on a verified domain.
  </FaqItem>
  <FaqItem title="CLOUDFLARE_ACCOUNT_ID">
    **Repo:** api. **Required:** when provider is cloudflare.
  </FaqItem>
  <FaqItem title="CLOUDFLARE_EMAIL_API_TOKEN">
    **Repo:** api. **Required:** when provider is cloudflare. Scope `Email Sending: Edit`.
  </FaqItem>
  <FaqItem title="RESEND_API_KEY">
    **Repo:** api. **Required:** when provider is resend.
  </FaqItem>
  <FaqItem title="SENDGRID_API_KEY">
    **Repo:** api. **Required:** when provider is sendgrid.
  </FaqItem>
  <FaqItem title="SMTP_HOST">
    **Repo:** api. **Required:** when provider is smtp. Use `mailpit` against the dev overlay; any RFC 5321 server in prod.

  <FaqItem title="SMTP_PORT">
    **Repo:** api. **Required:** when provider is smtp. Default 25; Mailpit listens on 1025.
  </FaqItem>
  <FaqItem title="SMTP_USER / SMTP_PASS">
    **Repo:** api. **Required:** optional. Auth when the server requires it.
  </FaqItem>
  <FaqItem title="NOTIFICATION_SETTINGS_URL">
    **Repo:** api. **Required:** no. Optional http(s) link rendered in notification email footers.
  </FaqItem>
</FaqGroup>

### Multi-tenant

<FaqGroup>
  <FaqItem title="ACCOUNT_DOMAIN_CLAIMING" open>
    **Repo:** api. **Required:** no (defaults `false`). When `true`,
    the first verified signup with a non-public email domain claims
    that domain on its personal account; subsequent signups from the
    same domain hit `DOMAIN_CLAIMED` (409) and must be invited
    through the standard invitations flow. The public-email allowlist
    (`gmail.com`, `outlook.com`, etc.) is hard-coded. Those domains
    always get fresh personal accounts. Leave off for consumer
    products. See [Multi-tenant model → Domain claiming](/api/multi-tenant/#domain-claiming-optional-b2b-mode).
  </FaqItem>
</FaqGroup>

### AI

<FaqGroup>
  <FaqItem title="AI_ENABLED" open>
    **Repo:** api. **Required:** no.
  </FaqItem>
  <FaqItem title="AI_PROVIDER">
    **Repo:** api. **Required:** when AI is on. One of `openai`, `anthropic`, or `noop`.
  </FaqItem>
  <FaqItem title="OPENAI_API_KEY">
    **Repo:** api. **Required:** when AI provider is openai.
  </FaqItem>
  <FaqItem title="OPENAI_BASE_URL">
    **Repo:** api. **Required:** no. Point at OpenAI-compatible APIs (OpenRouter, Ollama, vLLM).
  </FaqItem>
  <FaqItem title="OPENAI_DEFAULT_HEADERS">
    **Repo:** api. **Required:** no. Optional JSON object of extra headers passed on every OpenAI client request. Useful for OpenRouter ranking (`HTTP-Referer`, `X-Title`) or for custom auth on a compatible endpoint. Empty means no extra headers.
  </FaqItem>
  <FaqItem title="ANTHROPIC_API_KEY">
    **Repo:** api. **Required:** when AI provider is anthropic.
  </FaqItem>
</FaqGroup>

### UI build-time

<FaqGroup>
  <FaqItem title="VITE_API_URL" open>
    **Repo:** ui. **Required:** no (default empty). API base URL. Empty means same-origin (`/api/*` relative). Set a full HTTPS URL only for cross-origin deployments.
  </FaqItem>
  <FaqItem title="VITE_LOCALES">
    **Repo:** ui. **Required:** no. CSV of locale codes; first entry is the fallback.
  </FaqItem>
  <FaqItem title="VITE_GOOGLE_CLIENT_ID">
    **Repo:** ui. **Required:** no. Display gate for the Google button.
  </FaqItem>
</FaqGroup>

### Infra (compose-only)

<FaqGroup>
  <FaqItem title="STACK" open>
    **Repo:** infra. **Required:** yes. `dev` or `prod`.
  </FaqItem>
  <FaqItem title="POSTGRES_USER / POSTGRES_PASSWORD / POSTGRES_DB">
    **Repo:** infra. **Required:** yes. Postgres bootstrap credentials.
  </FaqItem>
  <FaqItem title="PUBLIC_UI_HOST">
    **Repo:** infra. **Required:** yes in prod. Apex DNS name for Traefik. Same-origin path routing serves both UI and `/api/*` from this single host.
  </FaqItem>
  <FaqItem title="ACME_EMAIL">
    **Repo:** infra. **Required:** yes in prod. Let's Encrypt contact address.
  </FaqItem>
  <FaqItem title="GRAFANA_ADMIN_USER / GRAFANA_ADMIN_PASSWORD">
    **Repo:** infra. **Required:** when observability overlay is on.
  </FaqItem>
  <FaqItem title="*_LIMITS_CPUS / *_LIMITS_MEMORY (and reservations)">
    **Repo:** infra. **Required:** no. Per-service resource caps; see [Resource limits](/infra/resource-limits/).
  </FaqItem>
  <FaqItem title="WITH_OBSERVABILITY / WITH_GLITCHTIP / WITH_BULLMQ / WITH_WUD">
    **Repo:** infra. **Required:** no. Overlay toggles.
  </FaqItem>
  <FaqItem title="RCLONE_REMOTE_NAME / RCLONE_REMOTE_PATH">
    **Repo:** infra. **Required:** when backups are configured.
  </FaqItem>
  <FaqItem title="BACKUP_RETENTION_DAYS">
    **Repo:** infra. **Required:** no. Default 30.
  </FaqItem>
  <FaqItem title="BACKUP_DRY_RUN">
    **Repo:** infra. **Required:** no. `1` means the script logs but does not act.
  </FaqItem>
</FaqGroup>
