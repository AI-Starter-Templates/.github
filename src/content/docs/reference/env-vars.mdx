Each repo has its own `.env.example` that's the **source of truth** for vars it consumes. This page is an index; it tells you which repo owns a given var, why it exists, and which `.env.example` to read for the full reference (defaults, comments, prod-only flags).

## Which file to edit

| Layer | File | Consumed by |
|---|---|---|
| **api-template** | `.env` in `api-template/` | The Bun process directly when running outside Docker |
| **ui-template** | `.env.local` in `ui-template/` | Vite at build/dev time (`VITE_*` only) |
| **infra (dev)** | `compose/.env` in `infra-docker-compose-template/` | Compose: injected into the api and ui dev containers |
| **infra (prod)** | `compose/.env` + `compose/api.prod.env` | Compose: prod profile uses separate api env file |

For Docker-Compose-driven setups (the default), **`compose/.env` is the single file you edit.** It overrides defaults baked into `docker-compose.yml` and gets injected into the right containers via `env_file:` references.

## Categories

### Process + observability

| Var | Repo | Required? | Notes |
|---|---|---|---|
| `NODE_ENV` | api | yes | `development` / `production` / `test`; gates prod-only invariants |
| `PORT` | api | no | Default 3000 |
| `LOG_LEVEL` | api | no | `debug` / `info` / `warn` / `error` |
| `SENTRY_DSN` | api | no | Empty → SDK is a no-op |
| `SENTRY_TRACES_SAMPLE_RATE` | api | no | 0..1, default 0.1 |
| `VITE_SENTRY_DSN` | ui | no | Same wire as Sentry/GlitchTip |

### Database + cache

| Var | Repo | Required? | Notes |
|---|---|---|---|
| `DATABASE_URL` | api | yes | Postgres connection string |
| `VALKEY_HOST` / `VALKEY_PORT` | api | yes for queues/cache | Defaults `localhost` / 6379 |
| `VALKEY_PASSWORD` | api | required in prod when Valkey-using features on | |
| `VALKEY_DB` | api | no | Default 0 |
| `CACHE_ENABLED` / `CACHE_PROVIDER` | api | no | `false` → noop; `memory` / `valkey` |
| `QUEUES_ENABLED` | api | no | When `false`, dispatch falls back to inline |

### Auth + session

| Var | Repo | Required? | Notes |
|---|---|---|---|
| `JWT_SECRET` | api | yes (≥ 32 chars) | Signs auth cookie; rotation invalidates all sessions |
| `FRONTEND_URL` | api | yes | OAuth success redirect target |
| `ALLOWED_ORIGINS` | api | yes in prod | CSV; must be HTTPS, no wildcards in prod |
| `GOOGLE_OAUTH_CLIENT_ID` / `_SECRET` | api | when Google enabled | |
| `GITHUB_OAUTH_CLIENT_ID` / `_SECRET` | api | when GitHub enabled | |
| `LINKEDIN_OAUTH_CLIENT_ID` / `_SECRET` | api | when LinkedIn enabled | |
